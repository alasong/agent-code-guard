# Agent Code Guard — 竞品比对报告

> 生成时间：2026-04-10 | 数据源：README 功能清单 + 源码实现分析

---

## 1. Agent Code Guard 定位

**一句话定位：** 轻量级 AI Agent 代码护栏 —— AST 静态分析 + 受限 builtins + 资源限制 + 审计日志，零依赖，同进程运行。

**核心特征：**
- **非沙箱** —— 明确声明 "不是安全沙箱"，定位是防意外而非防恶意
- **同进程执行** —— 通过 `exec()` + `SIGALRM` + `RLIMIT_AS` 实现隔离
- **零依赖** —— 纯 stdlib（pydantic 可选）
- **AST 预检** —— 执行前扫描 forbidden import / builtin / attribute
- **审计导出** —— JSON / CSV / Text 三种格式

## 3. 逐项对比

### 3.1 AST 静态分析

| 能力 | Agent Code Guard | RestrictedPython | E2B | Pyodide |
|------|:---:|:---:|:---:|:---:|
| 禁止 import 扫描 | ✅ | ✅ (语法转换) | ❌ (运行时隔离) | ❌ (无文件系统) |
| 禁止 builtin 扫描 | ✅ | ✅ | ❌ | ❌ |
| 危险属性检测 | ✅ (`__subclasses__` 等) | 部分 | ❌ | ❌ |
| 执行前拦截 | ✅ | ✅ | ❌ (执行后捕获) | ❌ |

**优势：** ACG 是唯一在纯 AST 层面同时覆盖 import / builtin / attribute 三维度的方案。

### 3.2 资源限制

| 能力 | Agent Code Guard | E2B | Docker | Pyodide |
|------|:---:|:---:|:---:|:---:|
| CPU 超时 | ✅ SIGALRM | ✅ 云服务配额 | ✅ cgroups | ✅ 浏览器限制 |
| 内存限制 | ✅ RLIMIT_AS | ✅ 云服务配额 | ✅ cgroups | ⚠️ WASM 堆大小 |
| 磁盘限制 | ❌ | ✅ | ✅ cgroups | ❌ (无磁盘) |
| 网络限制 | ❌ | ✅ | ✅ 网络命名空间 | ❌ |

**差距：** ACG 没有网络和磁盘限制能力，这是同进程方案的固有局限。

## 4. SWOT 分析

### Strengths
- **零依赖 + pip install** —— 最低门槛
- **同进程 ms 级启动** —— 适合高频短任务
- **内置审计 + 导出** —— 竞品普遍缺失
- **分级违规 (low/medium/high/critical)** —— 精细控制

### Weaknesses
- **软隔离** —— 无法防止 C 扩展逃逸、侧信道攻击
- **Unix-only** —— SIGALRM/RLIMIT 不支持 Windows
- **无网络/磁盘限制** —— 同进程方案固有局限
- **资源限制在同进程内不够可靠** —— `RLIMIT_AS` 对某些 Python 分配不生效

### Opportunities
- **AI Agent 生态爆发** —— Claude Code、LangChain、CrewAI 等框架需要轻量护栏
- **可集成为插件** —— 作为 LangChain Tool / MCP Server 提供
- **增加白名单机制** —— 允许项目自定义允许模块列表（当前只有黑名单）
- **Windows 兼容** —— 用 `threading.Timer` 替代 SIGALRM

### Threats
- **E2B 降价** —— 如果云端沙箱成本降至可接受范围，软隔离方案会被替代
- **Python 3.14+ 新安全特性** —— 如果 CPython 原生增加沙箱能力，第三方方案价值下降
- **大模型直接生成安全代码** —— 模型能力提升可能减少对护栏的需求

## 5. 差异化建议

| 优先级 | 建议 | 理由 |
|:---:|------|------|
| P0 | 增加白名单模式 (`allowed_modules`) | 黑名单永远有遗漏，白名单更安全 |
| P1 | 增加网络调用拦截 (monkey-patch socket) | 弥补同进程方案最大安全缺口 |
| P1 | LangChain Tool 适配器 | 直接对接最大用户群 |
| P2 | Windows 兼容 (threading.Timer 超时) | 扩大适用平台 |
| P2 | MCP Server 封装 | 对接 Claude Code 生态 |
| P3 | 异步支持 (asyncio 超时处理) | 现代 Python 趋势 |

---

## 6. 结论

Agent Code Guard 在 **"轻量护栏"** 这个细分赛道没有直接竞品：

- **对比 E2B/Docker：** 牺牲安全换极致简单 —— 适合不需要多租户隔离的场景
- **对比 RestrictedPython：** 更全面的检测维度（import+builtin+attribute）+ 审计日志
- **对比 Pyodide：** 服务端可用、无 WASM 开销

**核心竞争壁垒：** 零依赖 + 内置审计 + 分级违规。这三个特性组合在一起，当前市场没有完全相同的产品。

**最大风险：** 安全等级只有 ⭐，如果用户误将其当作安全沙箱使用，会产生责任问题。README 中 "Not a security sandbox" 的声明很好，但建议在 `pip install` 后的首次 import 时也打印此警告。
